Projects

Life-Saver (Web Application)

Objective: The ‘Life Saver’ application offers a multitude of features and services, primarily centered around organ and blood donation services. This application facilitates blood donations, allowing donors to both give and receive blood, while also organizing blood campaigns. In the event of an emergency requiring blood transfusion, this application proves particularly useful, ensuring that patients receive the necessary blood type. Additionally, the application offers organ donation services. It also provides information on the nearest blood bank and their current stock of blood, making it easier for people to find the required blood bank. Shortly, we plan to integrate a fundraising module, which will allow us to raise funds and allocate them to those in need of medical treatment but who lack the necessary financial means.

Technologies Used:
Front-End: HTML, CSS, JavaScript, XML
Back-End: Java, PHP, Firebase

Key Features:

  • Web Application Features: The web application is mainly focused on blood banks and admin management because managing the platform from a small screen is not feasible. Hospitals and blood banks typically manage their departments using computers.
    • Supports Light & Dark theme.
    • Uses MD5 hashing for secure password storage.
    • Protects against SQL Injection attacks.
  • Android Application Features: The Android app is designed for users, as Android is widely used worldwide, and mobile apps are more convenient for everyday use than a website.
    • Tip of the day feature.
    • Search & Voice Search functionality.

Outcome: The application streamlined the process of managing student data, reducing the time needed for data retrieval and updates.

Traveling-Site (Mini Project - Online Travel Platform)

Objective: "Let's Travel" is a PHP website where users can view information about various places uploaded by the admin. Users can access details about these places, and the admin has the ability to manage the entire website. This includes uploading information about new places, managing existing listings, and overseeing data related to the contact page, such as contact information and mobile numbers. The website serves as a platform for users to explore different travel destinations while giving the admin full control over the content and site management.

Key Features:

  • User Features:
    • View Places: Users can browse through a list of travel destinations/places uploaded by the admin. Each place is presented with detailed information, such as description, images, ratings, and possibly reviews from other users.
    • Explore Details: Users can click on a specific place to view more detailed information, including tourist attractions, accommodations, local cuisine, activities, and other relevant details.
  • Admin Features:
    • Place Management: Admins can add, edit, or delete travel destinations, providing detailed information about the place, such as descriptions, images, location details, and special attractions.
    • Contact Page Management: Admins can manage contact information, including full name, email, and phone numbers, ensuring users can easily reach out to website administrators.
    • Overall Website Management: Admins have access to the backend of the website, enabling them to perform various site management tasks, ensuring smooth and secure operations.

Technologies Used:
Front-End: HTML, CSS, JavaScript
Back-End: PHP, MySQL

Outcome: The "Traveling-Site" project allowed me to apply my web development skills to create a functional online platform. It helped me enhance my understanding of front-end design and basic back-end functionality, setting the stage for more advanced projects.
